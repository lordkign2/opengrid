# OpenGrid iOS Client

> Path: `ios/` | Language: Swift | UI: SwiftUI | Package Manager: Swift Package Manager

---

## Overview

The iOS client is a Swift application built with SwiftUI that integrates with the OpenGrid Rust core engine via Swift bindings generated by [Diplomat](https://github.com/rust-diplomat/diplomat). It mirrors the architectural model of the Android client: a stateless SwiftUI view layer over an `ObservableObject` that interfaces exclusively with the Rust engine.

## Architecture

```
┌─────────────────────────────────────────┐
│             SwiftUI View                │
│         (ContentView.swift)             │
│                                         │
│  @ObservedObject var observer           │
│  • Text("Status: \(observer.status)")   │
│  • Button → observer.initializeEngine() │
│  • Button → observer.submitDummyEvent() │
└───────────────────┬─────────────────────┘
                    │ @ObservedObject / @Published
                    ▼
┌─────────────────────────────────────────┐
│         OpenGridObserver                │
│         (ObservableObject)              │
│                                         │
│  @Published var engineStatus: String    │
│  private var engine: OpenGridEngine?    │
│  private var node: NodeHandle?          │
│                                         │
│  • Dispatches to background queue       │
│  • Posts updates back to main queue     │
└───────────────────┬─────────────────────┘
                    │ DispatchQueue.global(qos: .background)
                    ▼
┌─────────────────────────────────────────┐
│   Diplomat Swift Wrappers / Bindings    │
│   (Generated + manual Swift classes)   │
│                                         │
│  class OpenGridEngine { ... }           │
│  class NodeHandle { ... }               │
└───────────────────┬─────────────────────┘
                    │ C-ABI (via @_silgen_name or @_cdecl)
                    ▼
┌─────────────────────────────────────────┐
│      libopengrid_ffi.a / .dylib         │
│         (Rust FFI crate)                │
│              ↓                          │
│       opengrid-core (Rust)              │
└─────────────────────────────────────────┘
```

## Key Files

### `OpenGridIOSApp.swift`

The app entry point. Instantiates the observer and injects it into the view hierarchy:

```swift
import SwiftUI

@main
struct OpenGridIOSApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView(observer: OpenGridObserver())
        }
    }
}
```

### `ContentView.swift`

A stateless SwiftUI view. Receives an `OpenGridObserver` and renders its published state:

```swift
import SwiftUI

struct ContentView: View {
    @ObservedObject var observer: OpenGridObserver

    var body: some View {
        VStack(spacing: 20) {
            Text("OpenGrid Status: \(observer.engineStatus)")
                .padding()
                .multilineTextAlignment(.center)

            Button("Initialize Engine") {
                observer.initializeEngine()
            }
            .padding()
            .background(Color.blue)
            .foregroundColor(.white)
            .cornerRadius(8)

            Button("Submit Dummy Event") {
                observer.submitDummyEvent()
            }
            .padding()
            .background(Color.green)
            .foregroundColor(.white)
            .cornerRadius(8)

            Spacer()
        }
        .padding()
    }
}
```

**Design note:** The view contains **zero** business logic. It is a pure representation of `observer.engineStatus`.

### `OpenGridObserver.swift`

The bridge between SwiftUI and the Rust engine. All engine interactions happen here:

```swift
class OpenGridObserver: ObservableObject {

    @Published var engineStatus: String = "Not initialized"

    private var engine: OpenGridEngine?
    private var node: NodeHandle?

    func initializeEngine() {
        DispatchQueue.global(qos: .background).async {
            self.updateStatus("Initializing engine...")
            self.engine = OpenGridEngine()
            if let engine = self.engine {
                self.node = engine.createNode(name: "iOS Node")
            }
            DispatchQueue.main.async {
                self.updateStatus("Engine initialized successfully")
            }
        }
    }

    func submitDummyEvent() {
        DispatchQueue.global(qos: .background).async {
            do {
                guard let node = self.node else {
                    throw NSError(domain: "OpenGrid", code: 1,
                                  userInfo: [NSLocalizedDescriptionKey: "Node not initialized"])
                }
                let eventData = "dummy_event_\(Date().timeIntervalSince1970)"
                    .data(using: .utf8)!
                let version = node.submitEvent(payload: eventData)
                DispatchQueue.main.async {
                    self.updateStatus("Event submitted. Version: \(node.currentVersion())")
                }
            } catch {
                DispatchQueue.main.async {
                    self.updateStatus("Event submission failed: \(error.localizedDescription)")
                }
            }
        }
    }

    private func updateStatus(_ status: String) {
        DispatchQueue.main.async {
            self.engineStatus = status
        }
    }
}
```

**Key design decisions:**

| Decision | Reason |
|---|---|
| `DispatchQueue.global(.background)` | Rust FFI calls are synchronous — they must not block the main thread |
| `DispatchQueue.main.async` for updates | `@Published` properties must be mutated on the main thread |
| Engine + node held as optionals | They are only valid after `initializeEngine()` completes |
| Error wrapping in `NSError` | Provides native Swift error handling semantics |

## Building the iOS App

### Prerequisites

| Tool | Version | Notes |
|---|---|---|
| Xcode | 15+ | Required for iOS 17 sim targets |
| macOS | 13 (Ventura)+ | Required for latest Xcode |
| Rust | stable | With iOS targets installed |
| diplomat-tool | 0.14 | For regenerating Swift bindings |

### 1. Install Rust iOS Targets

```bash
rustup target add aarch64-apple-ios          # Physical device
rustup target add aarch64-apple-ios-sim      # Apple Silicon simulator
rustup target add x86_64-apple-ios           # Intel simulator (legacy)
```

### 2. Compile the Rust FFI Crate for iOS

```bash
# Physical device
cargo build --target aarch64-apple-ios --release -p opengrid-ffi

# Apple Silicon simulator
cargo build --target aarch64-apple-ios-sim --release -p opengrid-ffi

# Intel simulator (if needed)
cargo build --target x86_64-apple-ios --release -p opengrid-ffi
```

### 3. Create an XCFramework (Recommended)

An XCFramework bundles multiple architectures and simulator slices into one artifact:

```bash
# Create a fat simulator library (for both Intel and Apple Silicon simulators)
lipo -create \
  target/aarch64-apple-ios-sim/release/libopengrid_ffi.a \
  target/x86_64-apple-ios/release/libopengrid_ffi.a \
  -output target/universal-sim/libopengrid_ffi.a

# Package as XCFramework
xcodebuild -create-xcframework \
  -library target/aarch64-apple-ios/release/libopengrid_ffi.a \
  -headers ffi/ \
  -library target/universal-sim/libopengrid_ffi.a \
  -headers ffi/ \
  -output ios/OpenGridFFI.xcframework
```

### 4. Generate or Update Swift Bindings

```bash
diplomat-tool swift ffi/src/lib.rs -o ios/Sources/OpenGridIOS/Generated
```

### 5. Open and Build in Xcode

```bash
cd ios
open Package.swift
```

In Xcode:
1. Select your target scheme (`OpenGridIOS`)
2. Choose a simulator or connected device
3. Press **⌘+B** to build, **⌘+R** to run

## Architectural Rules

Things that **must never** appear in the SwiftUI layer:

| ❌ Prohibited | ✅ Correct approach |
|---|---|
| CRDT logic | Handled entirely in Rust core |
| Background networking | Transport abstracted in Rust layer |
| State stored in `@State` | State is a snapshot from Rust via ObservableObject |
| FFI calls on main actor | All FFI dispatched via `DispatchQueue.global(.background)` |
| Business logic duplication | Engine is the single source of truth |

## Concurrency Notes

### Why `DispatchQueue` instead of `async/await`?

The current implementation uses `DispatchQueue` rather than Swift's structured concurrency (`async/await`) because:

1. Diplomat-generated FFI calls are **synchronous** — there is no way to `await` them
2. The Rust engine is not `Sendable` — it cannot be passed across concurrency domains without explicit unsafe annotation
3. `DispatchQueue.global` with a callback to `DispatchQueue.main` is a well-understood, battle-tested pattern for this use case

**Future migration path:** As Diplomat adds support for `@Sendable`-annotated types, the codebase can be migrated to use Swift actors.

### Thread Safety Model

```
Main Thread:
  - SwiftUI view rendering
  - @Published property mutations
  - DispatchQueue.main.async callbacks

Background Thread (global QoS .background):
  - All Rust FFI calls
  - Engine initialization
  - Event submission
```

## Troubleshooting

### Build error: `cannot find type 'OpenGridEngine' in scope`

- The Swift bindings have not been generated or are in the wrong target.
- Run `diplomat-tool swift ...` and verify the output is in `Sources/OpenGridIOS/Generated/`.
- Make sure the generated files are included in the SPM target.

### Linker error: `library not found for -lopengrid_ffi`

- The compiled Rust library has not been linked to the Xcode target.
- Add the `OpenGridFFI.xcframework` to the target's "Frameworks, Libraries, and Embedded Content".
- Alternatively, add the `.a` file directly to "Link Binary With Libraries" in Build Phases.

### App crashes with `EXC_BAD_ACCESS` on FFI call

- A raw pointer from Rust has been used after the Rust object was freed.
- Ensure the `OpenGridEngine` and `NodeHandle` objects are kept alive for the entire duration of any calls that use them.
- Do not call `destroy` functions manually while a method on the same object may be in flight.

### `@Published` mutation crash: "Publishing changes from background threads is not allowed"

- A `@Published` property is being set from a background thread.
- Always wrap state updates in `DispatchQueue.main.async { ... }`.
